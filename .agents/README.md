# 서브에이전트 운영 규칙

모든 작업은 기본적으로 아래 순서를 필수 워크플로우로 따른다.

1. 메인 에이전트가 사용자 요청을 받고 목표, 범위, 제약, 검증 기준이 포함된 계획을 작성한다.
2. `plan-verifier`가 계획이 사용자 의도와 제약에 맞는지 확인한다.
3. `executor`가 승인된 계획과 허용된 범위 안에서 작업을 실행한다.
4. `plan-inspector`가 실행 결과가 승인된 계획을 따랐는지 검사한다.
5. 메인 에이전트가 검사 결과와 작업 산출물을 최종 검토하고 사용자에게 응답한다.

공통 원칙:
- 메인 에이전트가 최종 책임과 최종 판단 권한을 가진다.
- 서브에이전트는 맡은 단계의 근거와 결과를 간결하게 남긴다.
- 사용자 제약, 금지 파일, 쓰기 범위, 검증 기준을 계획과 실행 전후에 확인한다.
- 승인된 범위를 벗어나는 작업은 멈추고 메인 에이전트에 보고한다.
